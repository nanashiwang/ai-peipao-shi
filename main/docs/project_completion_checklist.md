## 1. 第一性原理

本项目不是单纯的“AI 生成工具”，而是陪跑师服务闭环系统。核心目标是：

1. 降低陪跑师重复劳动：减少翻聊天记录、写周报、重复回复、手工登记。
2. 提升服务稳定性：让每个家庭都有档案、风险、待办、跟进记录。
3. 保证触达安全：AI 内容必须可审核，发送必须可追溯，错误必须可阻断。
4. 沉淀业务资产：SOP、话术、画像、周报、PBL 点评、发送日志形成可复用数据。
5. 支持试点扩容：先跑通单陪跑师/单设备，再扩到多陪跑师、多设备、多校区。

## 3. 对抗式审查测试清单

### 3.1 发送前

- [x] 空内容不能创建发送任务。
- [x] `RPA????????` 这类乱码不能创建或更新任务。
- [x] 目标会话不在设备负责列表时不能领取。
- [x] 同一内容短时间重复发送要拦截或提示。
- [x] AI 高风险内容必须人工审核。
- [x] 导入空内容、缺家庭编号、非法日期、重复消息、疑似乱码必须被拦截或报告。

### 3.2 发送中

- [x] 当前前台窗口不是企业微信时必须停止。
- [x] OCR 点错会话时标题校验必须阻断。
- [x] 搜索结果为空时不能盲点坐标。
- [x] dry-run 不能按 Enter，且必须清空输入框。
- [x] 被控端未打开 `allow_real_send` 时必须阻断真实发送。
- [x] 真实发送只允许白名单会话。

### 3.3 发送后

- [x] 成功任务必须记录 `sent` 和设备 ID。
- [x] 失败任务必须记录失败原因。
- [x] 发送截图可追溯。
- [x] 发送任务创建、确认真实发送、修改、领取、取消和回写必须有审计日志。
- [x] 旧任务不能被误处理。
- [x] 日志能区分 dry-run 与 real-send。
- [x] 周报发送任务必须反向回写周报状态，避免“已审核但是否已发不可见”。
- [x] 家长周报签收回执：家长端可确认已查看已审核周报，陪跑师端可见签收状态，周报重新编辑后自动清空旧签收。（已完成基础版）
