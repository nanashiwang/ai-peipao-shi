# 微信客服接入

目标链路：家长在个人微信进入企业的微信客服会话，服务端接收消息，复用现有 AI 回复 Prompt、知识库与安全规则，再通过微信客服 API 回复。该通道与现有企业微信会话存档/RPA 并行，互不替换。

## 1. 微信后台准备

1. 在微信客服管理后台创建客服账号并开启 API。
2. 在「开发配置」获取企业 ID 和微信客服 Secret。
3. 设置回调 URL、Token、EncodingAESKey：

```text
https://你的域名/api/wecom-kf/callback
```

回调必须使用公网可访问的 HTTPS 地址。Token 不超过 32 位；EncodingAESKey 固定 43 位。Secret、Token、EncodingAESKey 只放服务器 `.env`，不要提交到 Git。

## 2. 服务器配置

在 `main/.env` 增加：

```env
WECOM_KF_ENABLED=true
WECOM_KF_CORP_ID=<企业ID>
WECOM_KF_SECRET=<微信客服Secret>
WECOM_KF_TOKEN=<回调Token>
WECOM_KF_ENCODING_AES_KEY=<43位EncodingAESKey>
WECOM_KF_DEFAULT_OPEN_KFID=<可选，默认客服账号ID>
WECOM_KF_POLL_ENABLED=true
WECOM_KF_POLL_INTERVAL_SECONDS=5
WECOM_KF_DISPATCH_ENABLED=true
```

重建服务后，进入「企微会话 -> 微信客服（个人微信入口）」检查状态。首次可点击“立即同步”；正常收到回调后，系统会自动记录 `open_kfid` 并按游标增量拉取。

## 3. Prompt 与知识库

微信客服和原企业微信通道使用同一份配置：

```text
控制端 -> Agent配置 -> AI回复 Agent
```

- `system_prompt`：统一回复 Prompt，保存后同时作用于微信客服和企业微信。
- `启用知识库检索`：开启后，系统按当前家长消息和历史上下文检索知识库。
- `检索条数`：建议先保持 4-6 条。
- 知识库范围：通用 SOP 选“全部 Agent”；仅回复使用的口径选“AI回复 Agent”。

系统会额外注入不可被普通 Prompt 覆盖的渠道约束：微信客服中不提企微群、RPA、设备或内部系统；图片、语音、文件等非文本消息只入库，不自动猜测内容。

默认知识库会自动补入「微信客服渠道回复规范」，已有人工 Prompt 和知识条目不会被覆盖。

## 4. 消息与家庭绑定

首次收到某个 `external_userid` 时，系统会创建稳定的微信客服家庭编号，并保存客户昵称、`open_kfid`、最近主动消息时间和回复额度。

如果该客户已存在于家庭档案，可在「企微会话 -> 微信客服（个人微信入口）-> 微信客户与家庭绑定」直接选择已有家庭；也可通过接口重新绑定：

```http
PUT /api/wecom-kf/bindings/{binding_id}
Content-Type: application/json

{
  "family_id": "已有家庭编号",
  "display_name": "家长显示名"
}
```

绑定列表可通过 `GET /api/wecom-kf/bindings` 查看。

## 5. 发送约束

服务端在真实发送前强制检查：

- 家长最近 48 小时内主动发送过消息；
- 当前消息窗口回复数少于 5 条；
- 文本不超过 2048 字节；
- 任务绑定了正确的 `external_userid` 和 `open_kfid`；
- 高风险、敏感或不确定内容继续执行现有人工复核规则；
- `msgid` 按任务稳定生成，避免同一任务产生不同业务消息 ID。

微信接口返回成功表示已受理；如果后续收到 `msg_send_fail` 事件，系统会把对应任务和日志回写为失败。

## 6. 上线检查

1. 数据库迁移到 `20260712_0004`。
2. `GET /api/wecom-kf/status` 返回 `configured=true`。
3. 微信后台回调校验通过。
4. 家长从微信客服链接/二维码进入并发送一条文本。
5. 控制端能看到消息、客户绑定和 AI 回复记录。
6. 先把自动回复设为 `dry_run` 验证 Prompt 与知识库，再切换到 `real_send`。
7. 验证 48 小时窗口、5 条额度、高风险转人工和失败事件回写。
