# 企业微信「会话内容存档」接入落地清单

> 目标：用官方 API 稳定、合规、结构化地接入企业成员与外部联系人的一对一（及群聊）沟通记录，
> 替代截图 + OCR 的消息摄入。本文是「开通权限后照着做」的清单。

代码侧已就绪：`app/services/wecom_archive.py`（SDK 封装 + 解密 + 归一化）、
后端端点 `/api/wecom-archive/status`、`/api/wecom-archive/sync`、`WecomArchiveState`（seq 游标表），
以及后台自动增量同步任务。**你只需开通、放文件、填 `.env`、重建容器。**

---

## 一、在企业微信后台开通

1. 管理后台 → 安全与管理 → 管理工具 → **会话内容存档**（金融行业选办公版；
   要存**员工与外部客户**沟通须选**服务版**，约 450 元/账号/年，可先免费试用 1 个月）。
2. **开启范围**：把需要存档的陪跑师成员加入范围（单聊须本人在范围内；群聊只需 1 名群成员在范围内）。
3. **合规告知**：外部联系人侧由平台自动弹出存档提示，客户同意后才存档——告知义务由平台完成。
4. 记录 **corp_id**（企业 ID）与该功能的 **Secret**。

## 二、密钥与可信 IP

1. 本地生成 RSA 密钥对：
   ```bash
   openssl genrsa -out wecom_archive_private_key.pem 2048
   openssl rsa -in wecom_archive_private_key.pem -pubout -out wecom_archive_public_key.pem
   ```
2. 后台「会话内容存档」填入**公钥**（记住版本号，SDK 拉取时会带 `publickey_ver`）。
3. **可信 IP**：把服务器出口公网 IP（`107.172.96.31`）加入接口的可信 IP 白名单。

## 三、放置 SDK 与私钥

官方 Finance SDK 是 Linux C 库。**务必下载 x86 服务器 SDK v3.0（原生支持 OpenSSL 3.0）**——
我们容器是 Debian 13 / OpenSSL 3，选 v3.0 才能避免旧版 `libcrypto.so.1.1 not found` 的依赖回补。
下载后连同私钥放进 `main/config/`（该目录已挂载到容器 `/app/config`）：

```
main/config/libWeWorkFinanceSdk_C.so
main/config/wecom_archive_private_key.pem
```

`.gitignore` 已忽略 `config/*.so`、`config/*.pem`、`config/*_private_key*`，不会误入库。

## 四、填写 .env

```bash
cp main/.env.wecom.example main/.env   # 若已有 .env 则把相关键并进去
```

关键项（其余见模板注释）：

```
WECOM_ARCHIVE_ENABLED=true
WECOM_ARCHIVE_POLL_ENABLED=true
WECOM_ARCHIVE_POLL_INTERVAL_SECONDS=60
WECOM_ARCHIVE_CORP_ID=<企业ID>
WECOM_ARCHIVE_SECRET=<存档Secret>
WECOM_ARCHIVE_SDK_PATH=/app/config/libWeWorkFinanceSdk_C.so
WECOM_ARCHIVE_PRIVATE_KEY_PATH=/app/config/wecom_archive_private_key.pem
WECOM_ARCHIVE_SELF_USERIDS=<陪跑师的userid,逗号分隔>
```

**映射（把存档会话对到系统家庭）**——`WECOM_ARCHIVE_CONVERSATION_MAP` 是 JSON：
```
{"外部联系人userid或roomid": {"target_name": "许宝月", "family_id": "F001"}}
```
不配也能跑：单聊按对方 userid、群聊按 roomid 自动归档为 `WECOM_<名称>`。

## 五、重建并验证

```bash
docker compose up -d --build api
```

1. **SDK 依赖自检**（最容易出问题的一步）：
   ```bash
   docker exec main-api-1 ldd /app/config/libWeWorkFinanceSdk_C.so
   ```
   用 v3.0 SDK 时这里应全部 `=> found`、无缺失。仅当**误用了旧版 SDK**才会出现
   `libcrypto.so.1.1 => not found`：优先换回 v3.0；确需旧版时，在 `Dockerfile` 的
   `apt-get install` 行从 Debian bullseye 源补装 `libssl1.1` 后重建：
   ```dockerfile
   RUN echo "deb http://deb.debian.org/debian bullseye main" >> /etc/apt/sources.list \
       && apt-get update && apt-get install -y --no-install-recommends libssl1.1 \
       && rm -rf /var/lib/apt/lists/*
   ```
   （只有 `ldd` 报缺时才需要；报缺什么就补什么。）

2. **状态自检**：前端「企微会话」页顶部「会话内容存档」卡应显示 `已接入`；或：
   ```bash
   curl -s http://127.0.0.1:8000/api/wecom-archive/status
   ```
   `configured:true`、`missing:[]` 即配置完整。

3. **首次同步**：点卡片「立即同步」，或 `POST /api/wecom-archive/sync`。返回 `pulled/normalized>0` 且
   陪跑会话里出现真实消息即成功。之后后台任务会按 `POLL_INTERVAL` 自动增量拉取。

## 六、切换与边界

- 存档跑通后，它成为消息摄入的**唯一稳定来源**；原 RPA 剪贴板/OCR 消息同步可停用。
- 存档**只含开通之后**的消息，开通前的历史单聊拿不回来——越早开通越好。
- 消息带全局唯一 `msgid`（落到 `RawMessage.external_id`，天然去重）。
- **发送侧不变**：官方对外部联系人无自由 1:1 发送 API，发送仍走被控端；存档可用于把
  「发送后回读校验」从截图识别改为「等存档里出现该 msgid」，更可靠。
