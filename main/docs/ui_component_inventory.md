# 陪跑师效率系统 UI 组件清单

这份清单只描述组件和信息结构，不限定视觉风格。后续可以把它交给 UI 设计工具或其他 AI 做视觉稿。

## 一、陪跑师日常必需组件

### 1. 主导航

- 工作台
- 陪跑会话
- 家庭档案
- 审核发送
- 系统设置

设计目标：陪跑师不需要理解系统架构，只按日常动作进入页面。

### 2. 顶部状态栏

- 当前页面标题
- 本地服务状态
- 当前操作状态：加载中、完成、失败
- 全局动作：导入数据、载入样例、批量分析、识别打卡

设计目标：AI 生成较慢时，必须明确告诉用户系统正在处理，而不是像按钮失效。

### 3. 工作台

- KPI 卡片：高风险家庭、待审核内容、待发送任务、已确认周报
- 今日优先处理列表
- 最近 AI 输出摘要

设计目标：打开系统先知道今天要处理什么。

### 4. 陪跑会话

这是最核心页面，建议作为真实产品的主页面。

- 会话列表
- 登录/账号状态
- 当前家庭聊天窗口
- 消息气泡：家长消息、陪跑师消息
- 消息输入框
- 快速生成回复按钮
- 完整分析按钮
- AI 协助侧栏

设计目标：陪跑师在一个页面里完成“看消息、生成回复、审核、发送”。

### 5. AI 协助侧栏

- 当前家庭摘要
- 家庭画像摘要：沟通风格、关注点、建议动作
- 待审核回复编辑器
- 发送、保存、取消按钮
- 最近 AI 结果摘要

设计目标：AI 不应单独成为一个页面，而应贴着当前会话提供帮助。

### 6. 审核发送

- 待发送任务表
- 内容编辑 textarea
- 单条发送
- 批量发送
- 取消任务

设计目标：作为统一审核兜底页，用来处理批量内容和异常任务。

### 7. 家庭档案

- 家庭基础信息
- 学员阶段档案：年级、课程阶段、Unit 进度、PBL 次数、打卡完成率、下一里程碑
- 家长画像
- 沟通时间线
- 快捷 Agent 操作

设计目标：不是日常主入口，而是需要复盘或深入理解家庭时使用。

## 二、必要但应降级的组件

### 1. 企微会话登记

当前 RPA 暂停后，不应作为主导航入口。可以保留在系统设置或高级区域。

### 2. AI 周报

周报很有价值，但不是每条消息都要看的页面。建议作为家庭档案里的一个 Tab 或工作台待办。

### 3. 打卡/PBL

适合做成会话内标签或家庭档案子模块，不建议独立主入口。

### 4. 发送日志

主要用于排障，不适合陪跑师日常使用。建议放在系统设置或任务详情里。

### 5. 话术模板

运营或管理员会用，普通陪跑师不应频繁进入。建议归入系统设置。

## 三、当前不建议放在主界面的内容

- 原始 JSON
- RPA 调试说明
- 模板技术字段
- 多个 Agent 独立页面
- 大面积表格化家庭列表

这些内容会让陪跑师误以为自己在操作后台系统，而不是在处理家长沟通。

## 四、AI 生成速度说明

当前有两种生成方式：

- 快速生成回复：只调用一次轻量回复 Agent，实测约 21 秒。
- 完整分析：调用画像 Agent 和回复 Agent，原始版本实测约 54 秒。

速度慢的主要原因：

- 豆包 Seed-2.0-pro 单次响应本身较慢。
- 完整分析会串行调用多个 Agent。
- 传给模型的上下文越多，输入和推理时间越长。
- 前端刷新多个列表不是主瓶颈，主瓶颈是大模型接口。

后续可优化方向：

- 日常默认使用快速生成回复。
- 完整分析改为后台任务，不阻塞页面。
- 使用更快的模型或专门的小模型做回复。
- 对家庭画像做缓存，只有新消息达到阈值时再更新。
- 模型接口支持流式输出后，在页面先展示生成中内容。

## 五、建议交给视觉设计 AI 的核心组件

- 左侧主导航
- 顶部状态栏
- 工作台 KPI 卡片
- 今日优先处理列表
- 会话列表
- 聊天消息气泡
- 消息输入框
- AI 协助侧栏
- 家庭画像摘要卡
- 待审核回复编辑卡
- 任务审核表
- 空状态
- 加载状态
- 错误提示
- Toast 轻提示
- 移动端单列布局
