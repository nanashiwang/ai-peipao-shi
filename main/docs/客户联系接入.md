# 企业微信客户联系接入

目标链路：家长添加企业成员后，系统同步客户关系并自动建档；后续可发送新客户欢迎语，也可定时创建一对一企业群发任务。

## 1. 企业微信后台

在企业微信管理后台进入「客户联系」相关应用，确认需要接入的成员已开通客户联系能力，并取得该应用的 Secret。随后配置客户联系事件回调：

```text
https://你的域名/api/wecom-customer/callback
```

回调必须是公网可访问的 HTTPS 地址。Token 和 43 位 EncodingAESKey 在该回调配置中填写，并与服务器环境变量保持一致。

需要的接口权限包括：获取配置了客户联系功能的成员列表、获取客户列表、获取客户详情、发送新客户欢迎语、创建企业群发任务。

## 2. 服务器配置

在 `main/.env` 增加：

```env
WECOM_CUSTOMER_ENABLED=true
WECOM_CUSTOMER_CORP_ID=<企业ID>
WECOM_CUSTOMER_SECRET=<客户联系应用Secret>
WECOM_CUSTOMER_TOKEN=<客户联系回调Token>
WECOM_CUSTOMER_ENCODING_AES_KEY=<43位EncodingAESKey>
WECOM_CUSTOMER_SYNC_ENABLED=true
WECOM_CUSTOMER_SYNC_INTERVAL_SECONDS=300
WECOM_CUSTOMER_DISPATCH_ENABLED=true
WECOM_CUSTOMER_WORKER_INTERVAL_SECONDS=30
WECOM_CUSTOMER_WELCOME_ENABLED=false
WECOM_CUSTOMER_WELCOME_TEXT=您好，已经收到您的添加。后续课程安排、任务提醒和学习反馈会通过这里与您同步。
```

重建服务后，进入「企微会话 -> 企业微信客户联系」检查状态。也可以调用：

```text
GET  /api/wecom-customer/status
POST /api/wecom-customer/sync
GET  /api/wecom-customer/bindings
```

## 3. 客户关系与聊天正文

- `change_external_contact` 回调只通知添加、编辑和解除客户关系，不包含聊天正文。
- 聊天正文继续由会话内容存档读取；私聊会按客户 `external_userid` 和负责成员 `userid` 自动映射到客户联系家庭档案。
- 同一客户可由多个成员负责，系统分别保存绑定关系，不会删除历史消息。
- 解除关系后绑定会标记为 `removed`，不再创建群发任务。

## 4. 欢迎语与定时提醒

启用 `WECOM_CUSTOMER_WELCOME_ENABLED` 后，系统收到新增客户回调会立即使用 `welcome_code` 发送欢迎语。该 code 有效时间很短且只能调用一次，因此不要在 Prompt 中临时生成欢迎语。

控制端可以选择客户、计划时间和内容创建提醒。到期后服务端调用企业群发接口，并把任务改为 `pending_confirmation`：

```text
系统创建企业群发任务 -> 对应成员在企业微信客户端确认 -> 企业微信触达客户
```

接口成功不等于客户已收到。当前实现不会把 `pending_confirmation` 自动标记为 `sent`。

## 5. 能力边界

- 客户联系官方接口不能像微信客服一样，由服务端在普通私聊中直接连续回复。
- 企业群发任务必须由对应成员确认，适合任务安排、课程通知和周期提醒。
- 需要实时 AI 私聊回复时，读取使用会话内容存档，发送使用已登录成员企业微信的 RPA。
- 企业群发频率受企业微信后台规则和客户接收上限约束，最终以企业微信当前后台配置为准。
