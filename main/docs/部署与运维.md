# AI 陪跑师效率系统 · 部署与运维文档

> 面向「一个总控服务器 + 多台被控电脑」的落地部署。被控端用 Windows 企业微信自动回复，
> 总控台统一下发任务、监控设备。本文档可照着一步步做。

---

## 一、整体架构

```
┌──────────────────────── 服务器（总控台）─────────────────────────┐
│  FastAPI 后端 + 前端看板 + PostgreSQL                              │
│  • 注册设备、分配 token                                            │
│  • 任务按会话动态分配（claim）                                     │
│  • 看板「设备监控」实时看每台在线/企微健康/任务统计               │
│  • 一键生成被控端「接入包」供下载分发                              │
└───────────────────────────┬──────────────────────────────────────┘
            claim / 心跳 / 回写（HTTP，被控端主动出站）
        ┌───────────────────┼───────────────────┐
     ┌──┴──┐             ┌──┴──┐             ┌──┴──┐
     │电脑A│             │电脑B│             │电脑C│   各跑一个被控端脚本
     │企微 │             │企微 │             │企微 │   只领自己负责会话的任务
     └─────┘             └─────┘             └─────┘   定位走阿里 ARK 云端
```

**关键特性**：被控端是**主动连出去**的一方，不需要公网 IP / 端口映射，在任何 NAT 后面只要能上网访问到服务器即可。需要开放端口的只有服务器。

## 二、服务器部署（总控台）

### 前置：准备阿里 ARK（百炼）密钥
被控端的会话定位走阿里云端视觉模型，必须有。**部署时不用填，部署完在看板「系统设置」页在线配置即可。**
1. 打开 https://bailian.console.aliyun.com/ ，登录并开通「百炼大模型服务」
2. 「API-KEY」页创建 Key（`sk-...`）
3. 模型用 `qwen-vl-plus`（视觉定位，已验证可用）

> 拿到 key 先放着，部署好后填进看板。

### 方式 0：裸服务器一条命令（最省事）
全新 Linux 服务器，一条命令自动装 git/docker、拉代码、起服务：
```bash
curl -fsSL https://raw.githubusercontent.com/nanashiwang/ai-peipao-shi/main/main/deploy/bootstrap.sh | bash
```
（若 `raw.githubusercontent.com` 访问不了，改用下面方式 A。）

### 方式 A：已拉代码后一键部署
```bash
git clone https://github.com/nanashiwang/ai-peipao-shi.git && cd ai-peipao-shi/main
sudo bash deploy/docker-deploy.sh        # 自动装 Docker + 建库 + 起服务 + 健康检查
```
跑完后端在 `http://服务器IP:8000`。compose 用 PostgreSQL 持久化、容器崩溃自动重启。

环境隔离建议：
- 试点：保持默认 `APP_ENV=pilot`，使用 Docker PostgreSQL；控制端默认要求登录，首次打开「登录注册」页创建第一个账号，该账号自动成为超管；ARK 可部署后在看板「系统设置」填写。
- 正式：在 `.env` 中设置 `APP_ENV=production`、独立 `POSTGRES_PASSWORD`/`DATABASE_URL`、`ADMIN_AUTH_SECRET`、`ADMIN_USERNAME`、`ADMIN_PASSWORD`，首个账号如需限制校区可加 `ADMIN_CAMPUS_NAMES`，并提前准备 `config/ark.json`；启动时会拒绝 SQLite、缺失 ARK 密钥或缺失管理端鉴权配置。
- 本地开发：`APP_ENV=local` 可使用 `sqlite:///./coach_mvp.db`，不要把本地库或 `config/ark.json` 提交到仓库。

**ARK 密钥不在这步填** —— 打开看板 →「系统设置」→ 填 API-KEY + 模型名（`qwen-vl-plus`）保存即生效（会写入 `config/ark.json` 并立即生效）。

常用运维命令：
```bash
docker compose logs -f api        # 看日志
docker compose restart api        # 重启
docker compose down               # 停止
git pull && sudo bash deploy/docker-deploy.sh   # 更新代码后重新部署
```

### 方式 B：裸机 systemd 部署
若不想用 Docker，用 `deploy/deploy.sh`（venv + systemd + nginx）。ARK 密钥同样在看板「系统设置」配。

### 对外用 80/443 + 域名
在后端前面挂一层 nginx 反代到 `127.0.0.1:8000`，参考 `deploy/nginx.yihe.site.conf`。

## 三、添加被控端（核心流程）

整个被控端接入只需在看板点几下 + 对方双击运行：

**你（在总控看板）：**
1. 浏览器打开 `http://服务器IP:8000` → 左侧「设备监控」
2. 「添加设备」填：
   - 设备ID：如 `rpa-01`（每台唯一）
   - 名称：如 `重庆01`
   - 负责会话：逗号分隔，如 `一合学社,周爸爸`（这台机器只领这些会话的任务）
3. 列表里点该设备的「**下载接入包**」→ 得到 `device_rpa-01.zip`，包内含 `package_manifest.json` 和 `校验接入包.ps1`
4. 把 zip 发给对方（微信/U盘都行）；如需零安装体验，可先在 Windows 构建机运行 `rpa\build_signed_client.ps1` 生成 EXE 版接入包

**对方（被控电脑）：**
1. 装一次 Python 3.11（[python.org](https://www.python.org/downloads/)，**务必勾 Add Python to PATH**）
2. 解压 zip 到固定文件夹
3. 先运行 `校验接入包.ps1` 验证文件完整性
4. 企业微信 PC 端登录，停在要自动回复的会话界面（底部能看到输入框）
5. 脚本包双击「**启动.bat**」→ 首次自动建环境装依赖（1-2 分钟）→ 自动上线领任务；EXE 包双击「**启动EXE.bat**」
6. 你在看板看到该设备「在线」即接入成功
7. 需要长期值守时，先确认手动启动正常，再双击 `install_autostart.bat` 安装登录自启；取消时双击 `uninstall_autostart.bat`
8. 验证定位无误后，对方把 `rpa/config.json` 里 `allow_real_send` 改成 `true`，重启后才允许真实发送

> 接入包很小（约 30KB），对方机器只装 5 个轻依赖（pywinauto/pywin32/pyperclip/pillow/openai），**不含** paddleocr 那套重依赖——定位全走云端。

## 四、看板（总控台）功能

| 页面 | 作用 |
|------|------|
| 工作台 | KPI 总览、校区/陪跑师筛选、优先处理清单、最近 AI 输出 |
| 家长看板 | 家长账号只读查看本人家庭阶段进度、已审核周报和最近沟通，并可签收周报、提交满意度反馈 |
| 家庭档案 | 家庭列表、消息时间线、画像、周报 |
| 审核发送 | 待发送任务审核、编辑、发送；发送日志 |
| 管理看板 | 按陪跑师和校区统计服务质量、风险家庭、待审核、待发送和发送完成率，支持前端校区切换 |
| **设备监控** | 每台被控机在线/离线、企微健康、负责会话数、待发/已发/失败统计、下载接入包、SQLite 备份与非破坏式恢复演练（每 5 秒刷新） |
| 系统设置 | ARK 云端定位密钥、话术模板维护 |

课程阶段模板导入后，家庭详情会自动展示学员阶段档案：年级、校区、课程阶段、Unit 进度、PBL 次数、打卡完成率和下一里程碑。该模板可只更新家庭档案，不会强制生成聊天消息。家庭画像 Agent 生成后，会同步更新家长档案里的关注点、沟通风格、满意度、风险信号和续报意向。陪跑师可在家庭详情记录电话、私信、群提醒、周报、补课、投诉和续报沟通，这些跟进动作会进入家庭时间线、工作台待办和今日优先级。家长端低分周报反馈会自动生成需升级跟进。家庭详情 AI 操作区支持一键生成画像、周报、回复和打卡/PBL，并在同页复核审核稿。

AI 回复页的「自动生成待审回复」只批量生成 `ai_outputs` 审核草稿，跳过无有效家长消息或近 24 小时已有待审回复的家庭；它不创建发送任务，不调度设备，也不触发企业微信真实发送。

侧栏导航按使用频率分组：

1. 日常工作：工作台、陪跑会话、家庭档案、审核发送。
2. 资料与 AI：家庭/学员、AI画像、AI周报、AI回复、打卡/PBL。
3. 运营排障：管理看板、企微会话、发送日志、审计日志、设备监控、系统设置。

控制端发送任务按角色和状态分层：

1. 只读角色只能查看，接口同时返回脱敏视图。
2. 陪跑师可编辑、审核、发起企微 `dry-run` 和网页测试发送，但不能确认企业微信真实发送。
3. 超管可调度设备、创建控制端账号并确认 `real_send`，真实发送仍需要 Windows 被控端 `allow_real_send=true` 才会按发送键。
4. 已发送、已取消等终态任务只保留查看和审计，不再允许修改。

控制端页面状态统一：

1. 首次加载时，关键面板会显示加载态，避免白屏。
2. 数据为空时，会显示下一步动作说明，而不是只显示空表格。
3. 控制端请求失败时，会显示错误态和重试入口。
4. 工作台出现退费、投诉等高风险信号时，会显示风险态并引导进入管理看板。

移动端基础可用性：

1. 手机/窄屏下左侧栏切换为顶部横向滚动导航。
2. 工作台、家庭详情、聊天、审核发送等多栏页面自动单列堆叠。
3. 表格只在表格容器内部横向滚动，不拉宽整页。
4. 浏览器请求头里的中文操作人会编码传输，后端解码后再写审计日志。

## 五、备份与恢复演练

控制端「设备监控」页提供基础版数据备份：

1. 点击「创建备份」生成 `data/backups/coach_mvp_YYYYMMDD_HHMMSS.sqlite3`。
2. 点击「恢复演练」只做非破坏式校验：备份文件可读、核心表存在、`PRAGMA integrity_check` 通过。
3. 点击「下载」把备份文件保存到异地位置，避免服务器磁盘损坏时一起丢失。

> 备份文件会标记为 `raw_sensitive`，仍包含家长姓名、孩子信息、聊天内容和发送日志，只能由管理员加密留存；排障、演示或给外部协作方看数据时，优先使用 `/api/ops/redacted-export` 的脱敏快照。

控制端同页展示日志保留策略：

1. 默认发送日志保留 365 天、截图证据 90 天、运行日志 30 天，可通过 `.env` 的 `SEND_LOG_RETENTION_DAYS`、`SEND_SCREENSHOT_RETENTION_DAYS`、`RUNTIME_LOG_RETENTION_DAYS` 调整。
2. 清理前会展示过期对象数量；真正删除必须在控制端点击确认，不会在健康检查或备份流程里自动删除。
3. 运行日志只清理轮转文件（如 `server.log.1`），不会删除当前正在写入的 `server.log` / `server.err.log`。

> 当前基础版面向本地 SQLite MVP。Docker/PostgreSQL 正式环境仍建议配合 `pg_dump`、数据库卷快照和异地留存，后续再把 PostgreSQL 备份纳入控制端一键演练。

## 六、被控端运行要求（重要）

被控端要真实发消息，对方机器必须满足：
- 企业微信 PC 端**已登录、在前台、停在正常会话视图**（不是"已选择N条"的多选栏）
- 运行期间**不要操作鼠标键盘、不要切换窗口**（RPA 用键鼠操控企微，抢焦点会冲突）
- **联网**（每次定位会话调一次阿里云端，消耗你的 ARK 额度）
- `dry_run=true` 时只把内容粘到输入框、不真发；真实发送还必须把 `allow_real_send=true`
- 接入包内 `watchdog.ps1` 会守护 `wecom_sender.py`，异常退出后自动重启；日志写入 `logs/rpa-watchdog.log`

## 七、常见问题

**Q: 对方双击启动.bat 报"python 不是内部命令"**
→ Python 没装或没勾 Add to PATH。重装 Python 勾选该项。

**Q: 看板设备一直显示离线**
→ 检查被控端 `rpa/config.json` 的 `api_base_url` 是否为服务器**公网**地址；对方机器能否 `curl http://服务器IP:8000/health`；服务器防火墙/安全组是否放行 8000。

**Q: 发送前报"当前聊天标题不是「XXX」，已阻止发送"**
→ 这是**防发错群的安全闸门**，正常。说明定位到的会话不对，重试即可（保持企微在正确会话前台）。

**Q: 定位失败 / 发不出去**
→ 确认企微在前台、未进入多选模式（按 Esc 退出）、`config/ark.json` 的 ARK key 有效且有额度。

**Q: 杀毒报毒**
→ 优先用 `rpa\build_signed_client.ps1 -CertThumbprint <证书指纹>` 生成机构签名 EXE 包；无证书时可先 `-SkipSign` 验证流程，脚本包可对解压目录加信任。

**Q: git push 报 `Failed to connect ... via 127.0.0.1`**
→ 本机 git 配了走代理（如 `127.0.0.1:7897`），但代理软件没开。开代理后再 push；或改端口 `git config --global http.proxy http://127.0.0.1:<端口>`。

**Q: 旧数据库升级后缺 device_id 列报错？**
→ 后端启动会自动跑 `ensure_columns()` 给旧表补 `device_id` 列（见 `app/db.py`），无需手动迁移、不会删数据。

## 九、安全须知

- `config/ark.json` 含你的阿里密钥，已 gitignore；**接入包里也会带它**，发给对方=对方共用你的额度，**只发可信的人**。
- 每台设备的 token 在接入包内，等于这台机器的钥匙，别外泄。
- 被控端默认 `allow_real_send=false`，即使总控台任务是 `real_send` 也不会按发送键；必须由可信操作者在 Windows 端显式打开。
- 数据备份包含家长姓名、聊天内容、发送日志等敏感信息，下载后必须按机构数据安全要求加密保存。
- 管理端支持超管、陪跑师、只读角色；首次注册账号自动成为超管，后续控制端账号只能由超管创建。陪跑师只能访问本人负责的家庭及关联数据，账号绑定 `campus_names` 后只能访问对应校区，只读角色访问家庭、时间线、AI 输出、周报、画像、发送任务和发送日志时会自动脱敏。
- 家长端使用独立 `parent_token`，只能读取绑定家庭的阶段进度、已审核周报和最近沟通，并只能签收/反馈本人家庭的已审核周报，不复用控制端管理员权限。
- 对公网开放前必须确认管理端鉴权开启、正式环境显式配置 `ADMIN_AUTH_SECRET`，确认管理端限流开启，并使用 HTTPS。
